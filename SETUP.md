# Настройка синхронизации между устройствами (Firebase)

Без этой настройки приложение работает как раньше — список хранится
только в этом телефоне/браузере. Чтобы все участники видели один и тот
же список в реальном времени, нужно один раз (~5 минут) создать
бесплатную базу данных на Firebase и вставить несколько строк в код.

## Шаг 1. Создать проект Firebase

1. Открой https://console.firebase.google.com
2. Войди с любым Google-аккаунтом.
3. «Добавить проект» → введи любое имя, например `vnl-order` → далее,
   далее, создать проект (Google Analytics не нужен, можно отключить).

## Шаг 2. Включить Realtime Database

1. В меню слева: **Build → Realtime Database**.
2. «Создать базу данных».
3. Регион — любой ближайший (например `europe-west1`).
4. Режим запуска — выбери **«Начать в тестовом режиме»** (test mode).
   Это даст открытый доступ на 30 дней — на следующем шаге поставим
   правила без ограничения по времени.

## Шаг 3. Настроить правила доступа

Во вкладке **Rules** вставь:

```json
{
  "rules": {
    "vnlOrder": {
      ".read": true,
      ".write": true
    }
  }
}
```

и нажми «Опубликовать».

**Важно понимать:** это открытый доступ — любой, кто узнает адрес твоей
базы (`databaseURL`), сможет читать и менять список. Для внутреннего
инструмента маленькой команды это обычно приемлемый компромисс
(ссылку никто не публикует), но это не полноценная защита данных. Если
захочешь закрыть доступ по-настоящему — напиши, добавим вход по паролю
(Firebase Authentication) или ограничение по домену.

## Шаг 4. Получить конфигурацию

1. Слева вверху — значок шестерёнки → **Настройки проекта**.
2. Внизу раздела «Мои приложения» нажми иконку `</>` (веб-приложение).
3. Дай любое имя, «Зарегистрировать приложение».
4. Появится блок кода `const firebaseConfig = {...}` — скопируй его.

## Шаг 5. Вставить конфигурацию в приложение

Открой `index.html`, найди блок:

```js
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  databaseURL: "https://YOUR_PROJECT-default-rtdb.firebaseio.com",
  projectId: "YOUR_PROJECT"
};
```

и замени значениями из шага 4 (нужны именно эти четыре поля, остальные
из консоли — `storageBucket`, `messagingSenderId`, `appId` — можно не
добавлять, они не используются).

## Шаг 6. Залить на GitHub / GitHub Pages

Просто замени файлы в своём репозитории на новые (`index.html`,
`manifest.json`, `service-worker.js`, папку `icons/`). GitHub Pages сам
подхватит изменения при следующем деплое — отдельного бэкенда или
сервера поднимать не нужно, база данных живёт на стороне Firebase.

## Как проверить, что заработало

1. Открой сайт на телефоне A, поставь количество в любой позиции.
2. Открой тот же адрес на телефоне B — значение должно появиться
   само, без обновления страницы.
3. Наверху экрана должна появиться зелёная точка и надпись
   «Синхронизировано». Серая точка = локальный режим (конфиг не
   заполнен), жёлтая = сеть пропала, но изменения сохранятся и
   отправятся, когда связь вернётся.

## Если что-то не сработало

- Проверь, что скопировал `databaseURL` полностью, вместе с
  `https://` и `.firebaseio.com`.
- Открой консоль разработчика в браузере (обычно F12) — ошибки
  Firebase обычно понятно объясняют причину (неверный ключ, база не
  создана и т.п.).
- Правила доступа (шаг 3) должны быть именно опубликованы — иначе
  запись будет падать с ошибкой permission_denied.
